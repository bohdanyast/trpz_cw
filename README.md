1. Спочатку впевніться, що у вас встановлене середовище виконання Java, бажано не нижче 17, (Java Runtime Environment), перевіривши це за допомогою командного рядка і команди java -version
2. Потім завантажте репозиторій і запустіть .exe файл. Ви маєте побачити поле вводу, кнопку оновлення та адресний рядок.
3. Я створив окремий сервер для тестування помилок 404, 502 та 503. Оскільки сервер є створеним власноруч, то оброблювати з нього запити я буду за допомогою власного обробника, оскільки по суті, тут дуже простий html та css. Для тестування у вас є посилання на:
-	test.com/index.html – код 200, звичайна сторінка
- test.com/404 – код 404
-	test.com/502 – код 502
-	test.com/503 – код 503

Для дуже складних сайтів, парсера, звісно, недостатньо, тому я застосував утиліту webEngine.loadPage(), бо зараз сучасні сайти містять дуже багато нових технологій, які виходять за межі стандартних файлів.
